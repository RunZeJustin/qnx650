Workflow of image creation

The main tasks involved in using the IDE to get Neutrino up and running on your board are:


  • Creating a new QNX System Builder project for an OS image a QNX System Builder project for an OS or a flash image for your board. The process is very simple if a BSP exists for your board. If an exact match isn't available, you may be able to modify an existing BSP to meet your needs.
  • Building your project to create the image.
  • Downloading an image to your target the OS image to your board. You might do this initially to verify that the OS image runs on your hardware, and then again (and again) as you optimize your system.
  • Configuring your QNX System Builder projects your projects.
  • Optimizing all libraries in your image your system by reducing the size of the libraries.
Note: To create an EFS project, you'll need to follow the workaround instructions found in Creating a new QNX System Builder project for an OS image.
